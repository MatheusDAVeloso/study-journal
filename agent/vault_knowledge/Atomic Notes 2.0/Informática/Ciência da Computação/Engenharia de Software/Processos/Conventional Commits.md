# Conventional Commits

---
## Definição
Padronizar commits a fim de facilitar a leitura de mudanças por outros desenvolvedores.

---
## Padrões
`feat:` — indicam que o trecho do código está incluindo um novo recurso, relacionando com o MINOR do versionamento semântico.

`fix:` — indicam correções de problemas (bug fix), se relacionando com o PATCH do versionamento semântico.

`docs:` — indicam alterações em documentações, sem alterações em código.

`perf:` — indicam alterações relacionadas a aumento de performance

`refactor:` — indicam alterações que mudam parte do código sem alterar sua funcionalidade;

`cleanup:` — indicam remoção de código morto, trechos comentados desnecessários ou qualquer outra limpeza de código fonte.

`remove:` — indicam exclusão de arquivos, diretórios ou funcionalidades obsoletas ou não utilizadas, reduzindo o tamanho e complexidade do projeto.
